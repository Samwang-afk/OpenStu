# OpenStu v0.1.0-alpha

First Alpha release. Core Q&A loop with minimal daily UX, source retrieval, and provider setup.

## Features

- **Minimal Daily UX Shell** — top status line, conversation area, input box. No dashboard. No visible mode bar.
- **Startup Decision** — detects whether a subject has materials, a plan, due reviews, or a close deadline and shows a tailored first message or option box.
- **Ctrl+X Action Palette** — registry-driven global action surface. 11 actions: create/switch subject, add materials, configure provider, view progress, view sources, make plan, exam review, change style, help, quit.
- **Create / Switch / Add Materials** — interactive onboarding flows accessible via Ctrl+X. Name-based creation, index or name switching, file/URL import.
- **Basic Ask Flow** — type a question, receive an AI tutor response. Works with or without a selected subject. Direct or Socratic response style depending on question type.
- **Provider Setup** — guided multi-step setup via Ctrl+X. Supports OpenAI-compatible, Anthropic, Google Gemini, and Ollama. API keys masked during entry; session-only unless set via environment variables.
- **Materials Retrieval & Source Citations** — Ask uses imported materials as context when relevant. Source references displayed with title and page/section locator after responses. Retrieval scoped to current subject. Graceful fallback on search failure.

## Technical Summary

- **Runtime**: Bun >= 1.3
- **TUI**: @opentui/solid + SolidJS
- **AI**: Vercel AI SDK (OpenAI-compatible, Anthropic, Google, Ollama)
- **Database**: SQLite via bun:sqlite (local, WAL mode, FTS5)
- **Tests**: 64 passing (bun test)

## Install

```powershell
git clone https://github.com/Samwang-afk/OpenStu.git
cd OpenStu
bun install
bun run dev
```

Or build standalone:

```powershell
bun run build
.\dist\openstu.exe
```

## Known Limitations

- API keys entered in TUI are session-only; use environment variables for persistence
- Planning, formal assessment, and spaced review are not part of the Alpha default UX
- CJK source retrieval may fall back to substring search
- Windows-only tested; macOS/Linux untested
- No sidebars, no cloud sync, no telemetry
